Общая информация о веб-сервисе
ReportApi — это веб-сервис, предназначенный для автоматизированного формирования отчётов об успеваемости студентов на основе входных данных, передаваемых через HTTP-запросы. Сервис реализован в виде REST API и ориентирован на использование в образовательных и учебно-методических системах.

Решаемая задача и целевая аудитория
Основной задачей веб-сервиса является упрощение и автоматизация процесса формирования отчётов по результатам учебной деятельности студентов. Использование сервиса позволяет стандартизировать формат отчётных данных и снизить объём ручной работы при их подготовке.

Целевой аудиторией сервиса являются преподаватели и сотрудники образовательных организаций, а также разработчики, интегрирующие API в информационные системы управления учебным процессом.

Функциональные возможности
Веб-сервис предоставляет следующий набор функций:
- проверка доступности и работоспособности сервиса;
- формирование отчёта на основе переданных данных о студенте;
- предоставление примера отчётных данных для ознакомления с форматом ответа;
- экспорт сформированных отчётов для дальнейшего использования.

Архитектура и используемые технологии
ReportApi разработан на платформе ASP.NET Core с применением REST-архитектуры, что обеспечивает простоту интеграции и расширяемость сервиса. Для автоматического формирования и визуализации документации веб-API используется стандарт OpenAPI и инструмент Swagger.

Развёртывание веб-сервиса выполнено в облачной среде Amvera, что обеспечивает публичный доступ к API и возможность его тестирования. В текущей реализации сервис не использует базу данных и работает с передаваемыми входными данными в режиме реального времени.

Развёртывание и запуск сервиса
Для развёртывания веб-сервиса необходимо выполнить следующие шаги:
1. Склонировать репозиторий с исходным кодом проекта.
2. При необходимости скорректировать конфигурационный файл `amvera.yml`.
3. Выполнить развёртывание приложения в облачной платформе Amvera.
4. Перейти по публичному URL сервиса для доступа к веб-API.
5. Открыть интерфейс Swagger UI по адресу `/swagger` для просмотра и тестирования доступных эндпоинтов.
